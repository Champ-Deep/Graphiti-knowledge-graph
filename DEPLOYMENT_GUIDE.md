## Railway Deployment (Recommended)

Railway provides the easiest deployment with automatic HTTPS, environment management, and Neo4j support.

### Prerequisites
- GitHub account
- Railway account (https://railway.app)
- Credit card (for billing, starts at ~$5/month)

### Step 1: Prepare Your Repository

1. **Ensure your code is pushed to GitHub:**
   ```bash
   git add .
   git commit -m "Prepare for Railway deployment"
   git push origin main
   ```

2. **Verify required files exist:**
   - `requirements.txt` ✓
   - `api_server.py` ✓
   - `.env.example` ✓

### Step 2: Deploy to Railway

1. **Go to Railway.app** and sign in with GitHub

2. **Create New Project:**
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Choose your `Graphiti-knowledge-graph` repository
   - Railway will auto-detect Python

3. **Add Neo4j Database:**
   - In your project, click "New"
   - Select "Database" → "Neo4j"
   - Railway will provision a Neo4j instance
   - Copy the connection details (will be auto-populated in env vars)

4. **Configure Environment Variables:**
   - Go to your API service → "Variables"
   - Add these variables:

   ```env
   # Railway auto-provides NEO4J_* variables, but verify:
   NEO4J_URI=neo4j+s://your-railway-neo4j.railway.app:7687
   NEO4J_USER=neo4j
   NEO4J_PASSWORD=<auto-generated>

   # Add your custom variables:
   OPENAI_API_KEY=<your-openrouter-key>
   OPENAI_BASE_URL=https://openrouter.ai/api/v1
   MODEL_NAME=anthropic/claude-3.5-sonnet

   TEAM_DOMAINS=yourdomain.com,yourcompany.io

   # API Security
   API_KEY=<generate-random-secure-key>
   API_HOST=0.0.0.0
   API_PORT=8000

   # Google OAuth (for Gmail)
   GOOGLE_CLIENT_ID=<your-client-id>
   GOOGLE_CLIENT_SECRET=<your-client-secret>
   ```

5. **Configure Start Command:**
   - Go to "Settings" → "Deploy"
   - Set start command:
   ```bash
   uvicorn api_server:app --host 0.0.0.0 --port $PORT
   ```

6. **Deploy:**
   - Railway will automatically deploy
   - Wait 2-3 minutes for build and deployment
   - Your API will be available at: `https://your-app.railway.app`

### Step 3: Get Your Public URL

1. Go to "Settings" → "Networking"
2. Click "Generate Domain"
3. Your API is now live at: `https://your-project-name.up.railway.app`

### Step 4: Test Your Deployment

```bash
# Test health endpoint
curl https://your-project-name.up.railway.app/health

# Test query endpoint (with your API key)
curl -X POST https://your-project-name.up.railway.app/api/query \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"account": "test-account", "query": "Who are the contacts?"}'
```

**✅ Done!** Your API is now live and ready to connect to ElevenLabs.

## Render Deployment

Render offers a free tier and is great for testing.

### Prerequisites
- GitHub account
- Render account (https://render.com)

### Step 1: Prepare Repository

1. **Create `render.yaml` in your project root:**

```yaml
services:
  - type: web
    name: graphiti-api
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn api_server:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: PYTHON_VERSION
        value: 3.11
      - key: NEO4J_URI
        sync: false
      - key: NEO4J_USER
        sync: false
      - key: NEO4J_PASSWORD
        sync: false
      - key: OPENAI_API_KEY
        sync: false
      - key: OPENAI_BASE_URL
        value: https://openrouter.ai/api/v1
      - key: MODEL_NAME
        value: anthropic/claude-3.5-sonnet
      - key: API_KEY
        generateValue: true
      - key: TEAM_DOMAINS
        sync: false

databases:
  # Note: Render doesn't offer Neo4j, use Neo4j Aura separately
```

### Step 2: Deploy to Render

1. **Go to Render Dashboard** → "New" → "Blueprint"
2. **Connect GitHub** repository
3. **Configure:**
   - Select `render.yaml`
   - Add environment variables in dashboard
   - For Neo4j: Use Neo4j Aura (see [Neo4j Database Setup](#neo4j-database-setup))
4. **Deploy:**
   - Click "Apply"
   - Wait for deployment (~5 minutes)
   - Get your URL: `https://graphiti-api.onrender.com`

### Limitations on Free Tier
- Spins down after 15 min of inactivity (cold starts ~30s)
- 750 hours/month free
- Good for testing, not production

## DigitalOcean VPS Deployment

For full control and better performance at lower cost.

### Prerequisites
- DigitalOcean account
- Basic Linux knowledge
- SSH client

### Step 1: Create Droplet

1. **Create a Droplet:**
   - OS: Ubuntu 22.04 LTS
   - Plan: Basic ($12/mo - 2GB RAM recommended)
   - Data center: Closest to your users
   - Authentication: SSH keys (recommended) or password
   - Hostname: `graphiti-api`

2. **SSH into your server:**
   ```bash
   ssh root@your-droplet-ip
   ```

### Step 2: Install Dependencies

```bash
# Update system
apt update && apt upgrade -y

# Install Python 3.11
apt install -y python3.11 python3.11-venv python3-pip git nginx certbot python3-certbot-nginx

# Install Docker (for Neo4j)
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

### Step 3: Setup Application

```bash
# Create app user
adduser --disabled-password --gecos "" graphiti
su - graphiti

# Clone repository
git clone https://github.com/YOUR_USERNAME/Graphiti-knowledge-graph.git
cd Graphiti-knowledge-graph

# Create virtual environment
python3.11 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Step 4: Setup Neo4j with Docker

```bash
# Exit graphiti user, return to root
exit

# Run Neo4j container
docker run -d \
  --name neo4j \
  --restart unless-stopped \
  -p 7474:7474 -p 7687:7687 \
  -v neo4j_data:/data \
  -v neo4j_logs:/logs \
  -e NEO4J_AUTH=neo4j/your-secure-password-here \
  neo4j:5.26.2

# Verify it's running
docker ps
```

### Step 5: Configure Environment

```bash
# As graphiti user
su - graphiti
cd Graphiti-knowledge-graph

# Create .env file
cat > .env << EOF
# Neo4j
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your-secure-password-here

# OpenRouter/OpenAI
OPENAI_API_KEY=your-key-here
OPENAI_BASE_URL=https://openrouter.ai/api/v1
MODEL_NAME=anthropic/claude-3.5-sonnet

# API
API_KEY=your-secure-api-key-here
API_HOST=0.0.0.0
API_PORT=8000
TEAM_DOMAINS=yourdomain.com

# Google OAuth
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
EOF

chmod 600 .env
```

### Step 6: Setup Systemd Service

```bash
# Exit to root user
exit

# Create systemd service
cat > /etc/systemd/system/graphiti-api.service << EOF
[Unit]
Description=Graphiti Knowledge Graph API
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=graphiti
WorkingDirectory=/home/graphiti/Graphiti-knowledge-graph
Environment="PATH=/home/graphiti/Graphiti-knowledge-graph/.venv/bin"
ExecStart=/home/graphiti/Graphiti-knowledge-graph/.venv/bin/uvicorn api_server:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# Start service
systemctl daemon-reload
systemctl enable graphiti-api
systemctl start graphiti-api

# Check status
systemctl status graphiti-api
```

### Step 7: Setup Nginx Reverse Proxy

```bash
# Create nginx config
cat > /etc/nginx/sites-available/graphiti-api << EOF
server {
    listen 80;
    server_name your-domain.com;  # Replace with your domain

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
    }
}
EOF

# Enable site
ln -s /etc/nginx/sites-available/graphiti-api /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

### Step 8: Setup SSL with Let's Encrypt

```bash
# Get SSL certificate (replace with your domain and email)
certbot --nginx -d your-domain.com --email your@email.com --agree-tos --no-eff-email

# Auto-renewal is configured automatically
# Test renewal
certbot renew --dry-run
```

### Step 9: Configure Firewall

```bash
# Setup UFW firewall
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw enable
```

**✅ Done!** Your API is now running at `https://your-domain.com`

## AWS Deployment

For enterprise-grade deployment with auto-scaling.

### Architecture Overview
- **ECS Fargate**: Container orchestration (API)
- **RDS or EC2**: Neo4j database
- **ALB**: Load balancer with SSL
- **Route53**: DNS management
- **Secrets Manager**: Secure credential storage

### Quick Steps

1. **Create ECR Repository:**
   ```bash
   aws ecr create-repository --repository-name graphiti-api
   ```

2. **Build and Push Docker Image:**
   ```bash
   # Create Dockerfile (see configuration files section)
   docker build -t graphiti-api .
   docker tag graphiti-api:latest YOUR_ECR_URI:latest
   docker push YOUR_ECR_URI:latest
   ```

3. **Create ECS Cluster & Service:**
   - Use AWS Console or Terraform
   - Configure task definition with environment variables
   - Set up ALB for HTTPS

4. **Setup Neo4j:**
   - Option 1: Neo4j Aura (managed, recommended)
   - Option 2: EC2 instance with Neo4j

5. **Configure DNS:**
   - Point domain to ALB in Route53

**Note:** AWS deployment is complex. Consider using Railway/Render first, then migrate to AWS when you need scale.

### Required Environment Variables

```env
# === Neo4j Database ===
NEO4J_URI=bolt://localhost:7687          # or neo4j+s:// for SSL
NEO4J_USER=neo4j
NEO4J_PASSWORD=your-secure-password

# === LLM Provider (OpenRouter or OpenAI) ===
OPENAI_API_KEY=sk-or-v1-xxx              # Your API key

MODEL_NAME=anthropic/claude-3.5-sonnet    # or gpt-4, etc.

# === API Configuration ===
API_KEY=your-random-secure-api-key       # Generate with: openssl rand -hex 32
API_HOST=0.0.0.0
API_PORT=8000

# === Company Configuration ===
TEAM_DOMAINS=yourcompany.com,yourdomain.io  # Comma-separated

# === Google OAuth (for Gmail sync) ===
GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxx

# === Optional: Microsoft OAuth (for Outlook) ===
AZURE_CLIENT_ID=xxx
AZURE_CLIENT_SECRET=xxx
AZURE_TENANT_ID=xxx
```

### Generating Secure API Keys

```bash
# Generate a secure API key
openssl rand -hex 32

# Or in Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### Security Best Practices

1. **Never commit .env files** - Use `.gitignore`
2. **Use secrets managers** in production (AWS Secrets Manager, Railway encrypted vars)
3. **Rotate API keys** regularly
4. **Use environment-specific configs** (dev, staging, prod)
5. **Enable HTTPS only** in production
6. **Implement rate limiting** (add to API if needed)

## Neo4j Database Setup

You have several options for hosting Neo4j:

### Option 1: Neo4j Aura (Recommended for Production)

**Pros:** Managed, automatic backups, scaling, monitoring
**Cons:** More expensive (~$65/month minimum)

1. **Sign up at https://neo4j.com/cloud/aura/**
2. **Create database:**
   - Select region closest to your API
   - Choose plan (start with AuraDB Professional)
3. **Get credentials:**
   - Connection URI: `neo4j+s://xxxx.databases.neo4j.io`
   - Username: `neo4j`
   - Password: (provided on creation)
4. **Update .env:**
   ```env
   NEO4J_URI=neo4j+s://your-instance.databases.neo4j.io
   NEO4J_USER=neo4j
   NEO4J_PASSWORD=your-password
   ```

### Option 2: Docker Self-Hosted (Recommended for Development)

**Pros:** Free, full control
**Cons:** You manage backups, updates, scaling

```bash
# Create persistent volumes
docker volume create neo4j_data
docker volume create neo4j_logs

# Run Neo4j
docker run -d \
  --name neo4j \
  --restart unless-stopped \
  -p 7474:7474 -p 7687:7687 \
  -v neo4j_data:/data \
  -v neo4j_logs:/logs \
  -e NEO4J_AUTH=neo4j/your-password \
  -e NEO4J_PLUGINS='["apoc"]' \
  -e NEO4J_apoc_export_file_enabled=true \
  -e NEO4J_apoc_import_file_enabled=true \
  neo4j:5.26.2

# Access Neo4j Browser
# Open http://localhost:7474
```

### Option 3: Railway Neo4j (Easiest)

Railway provides one-click Neo4j deployment:

1. In Railway project, click "New"
2. Select "Database" → "Neo4j"
3. Environment variables are auto-configured
4. Cost: ~$10-15/month

### Backup Strategy

```bash
# Docker backup
docker exec neo4j neo4j-admin database dump neo4j --to-path=/backups

# Copy backup out
docker cp neo4j:/backups/neo4j.dump ./backup-$(date +%Y%m%d).dump

# Restore
docker exec neo4j neo4j-admin database load neo4j --from-path=/backups/neo4j.dump
```

## Troubleshooting

### API Won't Start

**Check logs:**
```bash
# Railway: View logs in dashboard
# Render: View logs in dashboard
# DigitalOcean: systemctl status graphiti-api
# Docker: docker logs container-name
```

**Common issues:**
- Missing environment variables → Check .env or platform settings
- Neo4j connection failed → Verify NEO4J_URI and credentials
- Port already in use → Change API_PORT
- Dependencies missing → Run `pip install -r requirements.txt`

### Neo4j Connection Issues

```bash
# Test connection
python3 << EOF
from neo4j import GraphDatabase

driver = GraphDatabase.driver(
    "bolt://localhost:7687",
    auth=("neo4j", "password")
)
driver.verify_connectivity()
print("✅ Connected!")
EOF
```

**Common fixes:**
- Check if Neo4j is running: `docker ps` or check Aura dashboard
- Verify URI format: `bolt://` for local, `neo4j+s://` for Aura
- Check firewall rules allow port 7687
- Ensure password is correct (no extra spaces)

### SSL Certificate Issues

```bash
# Check certificate
openssl s_client -connect your-domain.com:443

# Renew Let's Encrypt
certbot renew --force-renewal
```

### High Memory Usage

```bash
# Check memory
free -h
docker stats

# Optimize Neo4j memory (edit neo4j.conf)
dbms.memory.heap.initial_size=512m
dbms.memory.heap.max_size=1G
```

### API Returns 503

- Service not initialized → Neo4j connection failed during startup
- Check startup logs for errors
- Ensure Neo4j is accessible from API server

### Slow Response Times

1. **Enable query logging** in Neo4j
2. **Add caching** to API (Redis)
3. **Optimize Neo4j queries** with indices
4. **Scale up** instance size
5. **Use CDN** for static content

```bash
BACKUP_DIR=/home/graphiti/backups

# Neo4j backup
docker exec neo4j neo4j-admin database dump neo4j --to-path=/backups

EOF

# Add to crontab (daily at 2 AM)
crontab -e
0 2 * * * /home/graphiti/backup.sh
```

## Performance Optimization

### 1. Enable API Caching

Consider adding Redis for caching frequent queries:

```python
# Install: pip install redis
import redis
from functools import wraps

redis_client = redis.Redis(host='localhost', port=6379, decode_responses=True)

def cache_result(ttl=300):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            cache_key = f"{func.__name__}:{args}:{kwargs}"
            cached = redis_client.get(cache_key)
            if cached:
                return json.loads(cached)
            result = await func(*args, **kwargs)
            redis_client.setex(cache_key, ttl, json.dumps(result))
            return result
        return wrapper
    return decorator
```

### 2. Database Indexing

Ensure Neo4j indices are created:
```cypher
CREATE INDEX entity_name IF NOT EXISTS FOR (n:Entity) ON (n.name);
CREATE INDEX episode_time IF NOT EXISTS FOR (n:Episode) ON (n.valid_at);
```

### 3. Connection Pooling

Already configured in Graphiti client, but verify:
```python
# In GraphitiService
self.client = Graphiti(
    uri=self.neo4j_uri,
    user=self.neo4j_user,
    password=self.neo4j_password,
    max_connection_pool_size=50,  # Adjust based on load
)
```
